# Security Policy

## Supported Versions

The following versions of Operative are currently supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue, please report it responsibly.

### How to Report

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, report them via email to: **nihal.srivastava05@gmail.com**

Include the following information in your report:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes (optional)

### What to Expect

- **Initial Response**: We will acknowledge receipt of your report within 48 hours.
- **Status Updates**: We will provide updates on the progress of addressing the vulnerability at least every 7 days.
- **Resolution Timeline**: We aim to resolve critical vulnerabilities within 30 days of the initial report.

### After Reporting

- If the vulnerability is accepted, we will work on a fix and coordinate with you on disclosure timing.
- If the vulnerability is declined, we will provide an explanation of why it does not meet our criteria.
- We will credit reporters in the security advisory (unless you prefer to remain anonymous).

### Scope

This security policy applies to:

- The Operative Chrome extension
- The MCP demo server included in this repository
- Any official documentation or configuration files

### Out of Scope

- Third-party dependencies (please report to the respective maintainers)
- Vulnerabilities in Chrome or the Chrome AI APIs
- Social engineering attacks

## Security Best Practices

When using Operative:

- Only connect to trusted MCP servers
- Review agent configurations before deployment
- Keep the extension updated to the latest version
- Report any suspicious behavior immediately
